# Little Drafter — Codebase Discovery & Bootstrapping

This document outlines the architecture of the `craft-skeleton` and the necessary steps to bootstrap the `little-drafter` project from it.

## 🏆 Project Identity & Core Philosophy

**Little Drafter** is a desktop productivity application built on a modern, event-sourced architecture. It leverages the `craft-skeleton`, which provides a "vertical slice" pattern for rapid development of domain-rich features.

- **Architecture Style**: DDD + CQRS + Event Sourcing (via Emmett).
- **Deployment**: Local desktop app (Electron) with a local/remote MongoDB event store.
- **Framework**: `es-kit` — a custom internal framework built on top of [Emmett](https://event-driven-io.github.io/emmett/).

## 🛠️ Technology Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| **Desktop Shell** | Electron 27 | Cross-platform desktop environment, IPC management. |
| **Frontend** | React 18 + Vite 5 | Reactive UI, glassmorphism design system, Lucide icons. |
| **Backend** | Express + TypeScript | REST API for the frontend, command processing. |

## 🚀 Bootstrapping `little-drafter`

To initialize `little-drafter` from the `craft-skeleton`, follow these steps:

### 1. File Initialization
Copy the contents of `craft-skeleton/` into `little-drafter/`.
> [!TIP]
> You may want to exclude the `.git` directory and the example `backend-js/src/project` module if starting from a completely blank slate, though keeping `project/` as a reference is recommended.

### 2. Dependency Management
Run the following commands from the project root:
```powershell
# Install root dependencies (Electron, Vite, etc.)
pnpm install

# Install backend dependencies (Standalone package)
cd backend-js
pnpm install
```

### 3. Environment Configuration
Create a `.env` file in the root and/or `backend-js` directory (based on `.env.example`).
- `MONGODB_URL`: Connection string for the event store.
- `PORT`: Backend API port (default: 8000).

### 4. Verification (The First Run)
Ensure MongoDB is running locally, then execute:
```powershell
# One-click launch (Windows)
.\start.bat

# Manual launch (3 terminals)
# 1. Backend: cd backend-js; pnpm dev
# 2. Frontend: pnpm dev
# 3. Electron: pnpm start
```

## 📂 Key Architecture Patterns

### The `es-kit` Framework
Located in `backend-js/src/es-kit/`, this framework automates the boilerplate of event sourcing:
- **EntityRoot**: Base class for aggregates (Entities).
- **EntityCommandBus**: Routes commands to handlers and appends events to the store.
- **Projections**: Defines how events update read models (MongoDB collections or SQLite tables).

### Vertical Slice Pattern
Each feature/module is organized into:
1. **Domain**: Entity state transitions and event definitions.
2. **Application**: Command definitions and handlers.
3. **Projections**: Read models for the UI.
4. **API**: Express routes and controllers.
5. **Integration**: Side effects and cross-aggregate communication.
