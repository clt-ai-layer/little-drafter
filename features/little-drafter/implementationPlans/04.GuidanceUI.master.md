## 1. Feature Overview
- **Name**: Guidance UI
- **Goal**: Implement the toddler-friendly drawing interface with progressive reveal and pulse cues.
- **Complexity**: Moderate
- **Dependencies**: Catalog Service (for template data), Draft Domain (for state).

---

## 2. Implementation Phases

### Phase 1: Layout and Navigation
- Create `NavigationBar` component with large "Next" (Green) and "Back" (Orange) arrows.
- Implement bottom-bar layout (height `h-24` or similar).
- Wire up button clicks to dispatch `ProgressStep` and `UndoStep` commands (via hooks/service).

### Phase 2: SVG Rendering and Animation
- Implement `DrawingCanvas` to render the SVG from `CatalogService`.
- Apply visibility logic: `opacity: 1` if `stepIndex <= currentStepIndex`, else `opacity: 0`.
- Use `framer-motion` to apply a blue pulse (`scale: [1, 1.05, 1]`) to the group where `stepIndex === currentStepIndex`.

## 4. Unit Tests

### Test File: `packages/frontend/test/features/drawing/components/DrawingSession.test.tsx`

**Component Under Test**: `DrawingSession`

#### Test Groups:

##### Navigation
- **Given**: Start of session. **When**: Clicking "Next". **Then**: Dispatcher receives `ProgressStep`.
- **Given**: Middle of session. **When**: Clicking "Back". **Then**: Dispatcher receives `UndoStep`.

##### Visual Reveal
- **Given**: `currentStepIndex` is 1. **When**: Rendering. **Then**: Step 0 and 1 are visible, Step 2 is hidden.
- **Given**: A new step is revealed. **When**: Rendering. **Then**: The new step has the `animate` prop for pulsing.

**Mocks Required**: `useDraftSession` hook (mocking state and dispatch), `CatalogService` mock.
