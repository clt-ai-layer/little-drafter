## 2. Proposed Solution

The **little-drafter** feature implements a progressive SVG reveal system managed by an **Emmett Decider**. 

### Architecture Overview
- **Backend**: Emmett Decider (`DraftDecider`) manages session state (in-progress, completed) and step indexing. MongoDB stores the event stream and projections.
- **Frontend**: Next.js 15 application renders SVGs step-by-step. `framer-motion` provides visual pulse cues. Web Speech API provides TTS instructions.
- **Data Model**: SVGs are pre-segmented with metadata for reveal order and instructions.
