# Scenario Analysis: little-drafter

**Date**: 2026-07-06 | **Solution**: Guided Drawing Assistant | **Related**: 00.Instructions.master.md

---

## Solution Overview

A drawing session managed by an **Emmett Decider** (`Draft`) that guides a child through a multi-step SVG drawing. The child draws on paper, and the app reveals steps one-by-one with visual (blue pulse) and audio (TTS) cues.

### Key Changes from Current State

| Aspect | Current | After |
| ------ | ------- | ----- |
| Drawing Model | None | Progressive SVG Reveal |
| Navigation | N/A | Large Bottom Nav (Back/Next) |
| Feedback | N/A | Blue Pulse + TTS Instructions |
| State | N/A | Event Sourced (Draft Aggregate) |

---

## Scenario Categories

1. **Session Lifecycle** - Start, Progress, Completion.
2. **User Interaction** - Navigation, Replay, Undo.
3. **Guidance Quality** - Visual cues, Audio cues.
4. **Edge Cases** - Boundary conditions and rapid input.
5. **Robustness** - Failures and fallbacks.

---

## Scenario Analysis

### Category 1: Session Lifecycle

#### Scenario 1: Start New Drawing
**Example**: Child selects "Lion" from catalog.
- **Action**: `StartDraft(TemplateId: "lion")` command.
- **Outcome**: `DraftStarted` event emitted. State index set to 0. First group revealed.
**Fit**: ✅ **Perfect**

#### Scenario 2: Complete Drawing
**Example**: Child clicks "Next" on the final step of the drawing.
- **Action**: `ProgressStep` -> detects `index == maxSteps - 1`.
- **Outcome**: `DraftCompleted` event emitted. Achievement projection triggered.
**Fit**: ✅ **Enhanced** (Triggers celebration)

---

### Category 2: User Interaction

#### Scenario 3: Forward Progression
**Example**: Child finishes a step and clicks the large green arrow.
- **Action**: `ProgressStep` command.
- **Outcome**: `DraftStepProgressed` event. `CurrentStepIndex` increments.
**Fit**: ✅ **Perfect**

#### Scenario 4: Step Undo
**Example**: Child wants to see the previous line again and clicks the orange arrow.
- **Action**: `UndoStep` command.
- **Outcome**: `DraftStepUndone` event. `CurrentStepIndex` decrements.
**Fit**: ✅ **Perfect**

#### Scenario 5: Back from First Step
**Example**: Child clicks "Back" when on step 0.
- **Action**: `UndoStep` command.
- **Outcome**: State invariant prevents index < 0. No-op (or UI confirmation to exit).
**Fit**: ⚠️ **Acceptable** (Handled by Decider invariants)

---

### Category 3: Guidance Quality

#### Scenario 6: Visual Pulse Cue
**Example**: Step 3 is revealed.
- **Action**: Path group `group-3` rendered in blue.
- **Effect**: CSS animation `pulse-once` applied to group.
**Fit**: ✅ **Perfect**

#### Scenario 7: Audio Instruction (TTS)
**Example**: Step 4 starts: "Draw two small circles for ears."
- **Action**: UI reads `metadata.instruction` from group 4.
- **Effect**: `AudioService.speak()` triggered.
**Fit**: ✅ **Perfect**

---

### Category 4: Edge Cases

#### Scenario 8: Rapid Navigation
**Example**: Toddler clicks "Next" five times in one second.
- **Action**: Multiple `ProgressStep` commands.
- **Outcome**: Decider processes sequentially. Events emitted for each step. UI might skip animations but state remains consistent.
**Fit**: ✅ **Minimal** (Standard ES consistency)

#### Scenario 9: Single Step SVG
**Example**: A very simple template with only one reveal.
- **Action**: `StartDraft` -> Immediately reveals whole drawing. `Next` completes it.
**Fit**: ✅ **Perfect**

---

### Category 5: Robustness

#### Scenario 10: TTS Failure
**Example**: Browser Speech Synthesis not supported or fails.
- **Outcome**: `AudioService` catches error. Drawing continues with visual cues only.
**Fit**: ⚠️ **Acceptable** (Graceful degradation)

#### Scenario 11: Missing Template
**Example**: API requested for `TemplateId: "non-existent"`.
- **Outcome**: `CatalogService` returns `NotFound`. `DraftDecider` fails validation.
**Fit**: ✅ **Perfect** (Standard error handling)

## Risk Matrix

| Risk | Probability | Impact | Mitigation |
| ---- | ----------- | ------ | ---------- |
| SVG Segmentation Complexity | High | Medium | Manual grouping in design tool ensures quality control. |
| TTS Voice Quality | Low | Low | Use standard Web Speech API with fallback settings. |
| Persistence Latency | Low | Medium | Use optimistic UI updates for navigation. |

## Conclusion

**little-drafter** solution provides an **excellent** fit for all 11 identified scenarios. The Emmett Decider pattern perfectly handles the sequential nature of drawing steps, and the "Paper Support" pivot is fully addressed via visual/audio guidance.

**Key Benefits**: High accessibility for toddlers, robust state management, clear reward loop.
**Concerns**: Manual SVG prep time (mitigated by manual strategy decision).
