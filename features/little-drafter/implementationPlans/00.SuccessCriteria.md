# Success Criteria: little-drafter

## 1. Narrative End-State
When the **little-drafter** feature is complete, it will transform the drawing experience for young children (around age 4) by acting as a patient, encouraging guide. 

Imagine a child sitting at a table with a piece of paper and a crayon. On the screen, they see a simple, friendly interface with a large, colorful catalog of drawings. Upon selecting an item—say, a "Lion"—the app enters a focused drawing mode. The screen displays a clear, white canvas with only the first part of the lion (perhaps the head circle) visible. This new part pulses gently in a bright blue color, silently communicating "Look here, this is what we're drawing now." Simultaneously, a friendly voice speaks: "Draw a big circle for the lion's head."

The child follows the instruction on their paper. When they are ready for more, they find the controls right where they expect them: in a solid bottom bar, featuring a giant green arrow on the right. Tapping it triggers the next step. The blue pulse and a new verbal instruction ("Now draw two small ears on top") guide them further. If they miss something, a large orange arrow on the left lets them go back and see the previous step again.

As the final stroke is "revealed" on the screen and the child finishes their drawing on paper, a celebratory event is triggered. The app acknowledges their hard work with a visual reward (stars or a badge), and the backend updates their progress, building a sense of achievement and confidence in their creative skills. The app doesn't draw for the child; it empowers them to draw for themselves.

## 2. Happy-Path Walkthrough
1.  **Selection**: The user selects a template from the Drawing Catalog.
2.  **Session Start**: The application sends a `StartDraft(TemplateId)` command. 
    -   An `Emmett` session is initialized.
    -   A `DraftStarted` event is emitted.
    -   State: `Status: in-progress`, `CurrentStepIndex: 0`.
3.  **Visual Guidance**: The frontend renders the SVG group corresponding to `index 0`.
    -   The group receives a CSS pulse animation.
    -   The path is highlighted in blue.
4.  **Audio Guidance**: The `AudioService` (Web Speech API) reads the `instruction` metadata for the current step.
5.  **Progression**: The user taps the large green "Next" arrow.
    -   A `ProgressStep` command is sent.
    -   A `DraftStepProgressed` event is emitted.
    -   State: `CurrentStepIndex` increments to `1`.
    -   The next SVG group is revealed, pulses blue, and a new TTS instruction is played.
6.  **Completion**: The user taps "Next" on the final step.
    -   A `ProgressStep` command is sent.
    -   A `DraftCompleted` event is emitted.
    -   State: `Status: completed`.
7.  **Achievement**: The `AchievementProjection` processes `DraftCompleted`.
    -   The user's star/badge count is incremented in the MongoDB database.
    -   The UI displays a celebration/badge reward.

## 3. Acceptance Checklist
1.  **[AC-01]** The `StartDraft` command successfully initializes an Emmett Decider session with the correct `TemplateId` and `index 0`.
2.  **[AC-02]** The `CatalogService` correctly extracts `instruction` metadata from SVG path groups based on the `CurrentStepIndex`.
3.  **[AC-03]** The `DrawingSession` component reveals SVG paths progressively, ensuring only paths up to the `CurrentStepIndex` are visible.
4.  **[AC-04]** The latest revealed SVG group pulses in blue (`#0000FF`) exactly once upon reveal or navigation.
5.  **[AC-05]** The `AudioService` triggers a Text-to-Speech call using the Web Speech API whenever the step index changes.
6.  **[AC-06]** Clicking the large green "Next" arrow sends a `ProgressStep` command and updates the UI state.
7.  **[AC-07]** Clicking the large orange "Back" arrow sends an `UndoStep` command, decrements the index, and prevents negative indices via Decider invariants.
8.  **[AC-08]** Upon finishing the final step, the session status transitions to `completed` and the "Next" arrow triggers the celebration state.
9.  **[AC-09]** The `AchievementProjection` successfully updates the MongoDB `Achievements` collection in response to a `DraftCompleted` event.

## 4. Key Secondary Scenarios
-   **Back from Start**: If the user clicks "Back" while at `CurrentStepIndex: 0`, the state remains at `0` (invariant protection), and the UI provides a visual hint or prevents the action.
-   **Rapid Clicks**: If the user clicks "Next" multiple times rapidly, the Emmett Decider processes the commands sequentially, ensuring state consistency even if the UI skips intermediate animations.
-   **TTS Fallback**: If the Web Speech API is unavailable or fails, the application continues to function normally using visual pulse cues alone, without throwing a blocking error.
-   **Invalid Template**: If an invalid `TemplateId` is requested, the application displays a graceful "Template not found" error instead of crashing.

## 5. Explicit Out-of-Scope
-   **Digital Drawing**: The app does NOT provide tools for drawing on the screen (brushes, erasers, colors).
-   **Custom Uploads**: Users cannot upload their own SVGs in this iteration.
-   **Social Sharing**: Sharing drawings to social media or a gallery is deferred.
-   **Multi-child Sessions**: Simultaneous drawing sessions for multiple children on one device are not supported.
