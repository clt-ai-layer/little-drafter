## 1. Feature Overview
- **Name**: Draft Domain
- **Goal**: Implement the core state management for a drawing session using an Emmett Decider.
- **Complexity**: Moderate
- **Dependencies**: None (Core Domain)

---

## 2. Implementation Phases

### Phase 1: Events and Commands
- Define `DraftStarted`, `DraftStepProgressed`, `DraftStepUndone`, `DraftCompleted` events.
- Define `StartDraft`, `ProgressStep`, `UndoStep` commands.
- Ensure `@SerializableType` decorators are used with correct arity.

### Phase 2: Decider Logic
- Implement `DraftState` interface: `templateId`, `currentStepIndex`, `status` (`in-progress`, `completed`).
- Implement `initialState`.
- Implement `evolve()`: State transitions for all events.
- Implement `decide()`: 
    - `StartDraft`: Validate template, emit `DraftStarted`.
    - `ProgressStep`: Check if `currentStepIndex` is at max (emit `DraftCompleted`) or increment (emit `DraftStepProgressed`).
    - `UndoStep`: Check `index > 0`, emit `DraftStepUndone`.

**Component Under Test**: `DraftDecider`
**Test Pattern**: `DeciderSpecification` (Emmett pattern)

#### Test Groups:

##### Session Lifecycle
- **Given**: No state. **When**: `StartDraft`. **Then**: `DraftStarted`, status `in-progress`, index `0`.
- **Given**: `DraftStarted`. **When**: `ProgressStep` on final step. **Then**: `DraftCompleted`, status `completed`.

##### Navigation
- **Given**: `DraftStarted` (index 0). **When**: `ProgressStep`. **Then**: `DraftStepProgressed`, index `1`.
- **Given**: `DraftStepProgressed` (index 1). **When**: `UndoStep`. **Then**: `DraftStepUndone`, index `0`.

##### Invariants
- **Given**: `DraftStarted` (index 0). **When**: `UndoStep`. **Then**: No event (invariant protection).
- **Given**: `DraftCompleted`. **When**: `ProgressStep`. **Then**: No event (cannot progress finished draft).

**Mocks Required**: None (Pure functional logic).
