# Architecture Decision Record: little-drafter

## ADR-001: Emmett Decider for Drawing Sessions

**Status**: Accepted
**Date**: 2026-07-06
**Context**: The application needs to manage the sequential state of a drawing session (starting, progressing through steps, undoing, and completing). We need a robust, testable way to handle these transitions while preserving history and enabling future replay features.
**Decision**: Use an Emmett Decider pattern for the `Draft` aggregate.
**Consequences**: 
- **Pros**: Pure functional logic, easy to test via `DeciderSpecification`, naturally supports event sourcing, clear state transitions.
- **Cons**: Slightly higher boilerplate for simple state changes compared to traditional CRUD.
**Alternatives Considered**: 
- **Simple Mongo Document**: Rejected because it lacks the audit trail and state transition clarity needed for complex sequential guidance.

## ADR-002: Manual SVG Segmentation

**Status**: Accepted
**Date**: 2026-07-06
**Context**: We need to reveal SVG parts in a specific order with corresponding instructions. Automated segmentation of generic SVGs is technically complex and error-prone for the specific UX quality required for toddlers.
**Decision**: Use manual grouping (`<g>`) and tagging (`data-step`, `data-instruction`) in the SVG files.
**Consequences**:
- **Pros**: High quality control, predictable behavior, simple implementation on the frontend and backend.
- **Cons**: Initial asset preparation is a manual task for designers/developers.
**Alternatives Considered**:
- **Automated Path Analysis**: Rejected due to high implementation complexity and unpredictable reveal order.

## ADR-003: Dedicated Bottom Navigation Bar

**Status**: Accepted
**Date**: 2026-07-06
**Context**: 4-year-olds need very clear, stable interactive affordances. Overlays or small icons can be frustrating or difficult to discover.
**Decision**: Implement a dedicated, fixed bottom navigation bar containing only the "Next" and "Back" arrows.
**Consequences**:
- **Pros**: High accessibility, clear physical separation from the drawing canvas, stable interaction zone.
- **Cons**: Reduces available vertical space for the canvas.
**Alternatives Considered**:
- **Floating Action Buttons**: Rejected as they can obscure parts of the drawing.
