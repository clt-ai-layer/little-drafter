## 1. Feature Overview
- **Name**: Audio Guidance
- **Goal**: Provide verbal instructions for each drawing step using Text-to-Speech.
- **Complexity**: Simple
- **Dependencies**: Catalog Service (provides instruction text).

---

## 2. Implementation Phases

### Phase 1: Audio Service
- Implement `AudioService.ts` as a wrapper around `window.speechSynthesis`.
- Implement `speak(text: string)` method.
- Add configuration for voice selection (prefer natural-sounding voices if available) and rate.

### Phase 2: UI Integration
- Use an effect in `DrawingSession` to trigger `AudioService.speak()` whenever the `currentStepIndex` changes or when the session starts.

## 4. Unit Tests

### Test File: `packages/frontend/test/features/drawing/services/AudioService.test.ts`

**Component Under Test**: `AudioService`

#### Test Groups:

##### Speech Trigger
- **Given**: Instruction text. **When**: `speak`. **Then**: `speechSynthesis.speak` is called with a `SpeechSynthesisUtterance`.

##### Robustness
- **Given**: `speechSynthesis` is missing (unsupported browser). **When**: `speak`. **Then**: Logs a warning but does not throw.
- **Given**: Rapid triggers. **When**: `speak` called twice. **Then**: Cancels previous speech before starting new one.

**Mocks Required**: Global `speechSynthesis` and `SpeechSynthesisUtterance` mocks.
