# Implementation Plan: Draft Domain

## Task 1: Domain Implementation (Phase 7 - Task 1)

This plan covers the implementation of the core Draft domain logic using the Emmett/ES-Kit pattern discovered in the codebase.

### 1. Plan Details
- **Target Component**: `DraftDecider` (using `EntityRoot` pattern)
- **Target Files**:
    - `packages/backend/src/Draft/Domain/DraftEvents.ts`
    - `packages/backend/src/Draft/Domain/DraftCommands.ts`
    - `packages/backend/src/Draft/Domain/DraftDecider.ts`
    - `packages/backend/test/Draft/Domain/DraftDecider.test.ts`
- **Patterns**: Emmett `EntityRoot` for the aggregate, `Outcome` for result handling, and `vitest` for testing (based on skeleton).

### 2. Scope Analysis
- **Events**: `DraftStarted`, `DraftStepProgressed`, `DraftStepUndone`, `DraftCompleted`.
- **Commands**: `StartDraft`, `ProgressStep`, `UndoStep`.
- **Decider Logic**:
    - `evolve` (via `when` bridge) for state transitions.
    - `decide` (domain methods) for business logic and validation.
- **Invariants**:
    - Cannot undo past step 0.
    - Cannot progress a completed draft.
    - `StartDraft` only allowed on `Empty` lifecycle.

### 3. Key Considerations
| Area | Consideration |
| --- | --- |
| **SerializableType** | Must use arity 3 for events and 2 for commands/VOs as per `core.md` and `feature-creation-workflow.md`. |
| **Result Pattern** | Use `Outcome` (from `es-kit`) for domain methods; never throw. |
| **Testing** | Use `vitest` as seen in `craft-skeleton`; follow `DeciderSpecification` logic. |

### 4. Per-Output Content Plan
- **DraftEvents.ts**: Define event interfaces and schemas with `@SerializableType`.
- **DraftCommands.ts**: Define command interfaces and schemas with `@SerializableType`.
- **DraftDecider.ts**: Implement `DraftDecider` class extending `EntityRoot`. Include `lifecycle` state and domain methods.
- **DraftDecider.test.ts**: Unit tests covering lifecycle, navigation, and invariants.

### 5. Implementation Steps
1. Create `DraftEvents.ts` and `DraftCommands.ts`.
2. Create `DraftDecider.ts` implementing the state transitions and logic.
3. Create `DraftDecider.test.ts` and verify logic with tests.
