# Implementation Plan: Catalog Service

## Task 1: Plan (Phase 7 - Task 1)

This plan covers the implementation of the `CatalogService` for loading and parsing SVG templates with step-by-step instructions.

### 1. Plan Details
- **Target Component**: `CatalogService`
- **Target Files**:
    - `packages/backend/src/Catalog/Domain/ValueObjects/DrawingTemplate.ts`
    - `packages/backend/src/Catalog/Application/CatalogService.ts`
    - `packages/backend/test/Catalog/Application/CatalogService.test.ts`
- **Patterns**: Use `Outcome` from `@src/es-kit/domain/Outcome.js` for result handling. Use JSDoc for documentation.

### 2. Scope Analysis
- **DrawingTemplate VO**:
    - Represents a drawing with its steps.
    - Fields: `id`, `name`, `steps: DrawingStep[]`.
    - `DrawingStep`: `index`, `pathData`, `instruction`.
- **CatalogService**:
    - `getTemplate(id: string): Promise<Outcome<DrawingTemplate>>`
    - `listTemplates(): Promise<Outcome<{ id: string, name: string }[]>>`
    - Needs to read SVGs from a directory (e.g., `packages/backend/assets/templates`).
- **SVG Parsing**:
    - Use `jsdom` (if available) or a robust regex to find `<g>` with `data-step`.
    - Extract `data-instruction`. Fallback: "Follow the lines to draw the next part."
    - Aggregate all `<path>`, `<circle>`, `<rect>`, etc., within the group into a single string or array.

### 3. Key Considerations
| Area | Consideration |
| --- | --- |
| **Storage** | For now, templates will be stored in `packages/backend/assets/templates` as `.svg` files. |
| **Parsing** | Ensure `data-step` is treated as a number and steps are sorted. |
| **Missing Metadata** | Provide default instruction if `data-instruction` is missing. |
| **Testing** | Use `vitest` (or `jest` as per `core.md`, I'll check which one is configured). |

### 4. Per-Output Content Plan
- **DrawingTemplate.ts**: Define `DrawingTemplate` and `DrawingStep` types/classes with `@SerializableType`.
- **CatalogService.ts**: Implement service with `fs/promises` to read files and parsing logic.
- **CatalogService.test.ts**: Tests with mock filesystem or temp files.

### 5. Implementation Steps
1. Create `DrawingTemplate.ts` VO.
2. Create `CatalogService.ts` with basic listing/loading logic.
3. Implement SVG parsing within `CatalogService.ts`.
4. Create `CatalogService.test.ts` and verify.
