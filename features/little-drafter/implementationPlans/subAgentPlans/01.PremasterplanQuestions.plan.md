# Plan — Phase 1: Premasterplan (Questions) for little-drafter

## Overview
This plan covers the initial analytical phase for the **little-drafter** feature. The goal is to scan the available architectural context (DocRouters), analyze the specific requirements (especially the 4-year-old on-paper drawing guidance), and formulate a set of clarification questions to resolve ambiguities before implementation planning.

## Findings & Initial Analysis
- **Core Requirement**: A "craft" application assisting 4-year-olds in drawing by progressively revealing SVG parts on click.
- **Clarification**: The child draws on **PAPER**, not the screen. The app serves as a visual guide.
- **Architecture**: DDD + CQRS + Event Sourcing using **Emmett** and `es-kit` (functional deciders).
- **Technology**: Electron, React, Express, MongoDB.
- **DocRouter Status**: `_GlobalAgentDocRouter.md` not found; using `_RulesDocRouter.md` and `_EmmettDocRouter.md` as primary context seeds.

## Task 2: Formulate Questions (00.Questions.little-drafter.md)
I will group questions into the following dynamic sections:

### Section 1: Guidance Mechanics (The "Paper Support" Pivot)
- **Q1: Visual Representation of Steps.** How should the "next" line be distinguished from already revealed lines? (e.g., flashing, different color, or just appearing).
- **Q2: Contextual Awareness.** Since we don't have on-screen tracing, how does the app know the child is ready for the next step? Is a simple "Click to Continue" sufficient for a 4-year-old?
- **Q3: Guidance Cues.** Do we need audio instructions (e.g., "Now draw a big circle") or just visual arrows?

### Section 2: Domain & State (Emmett/ES)
- **Q4: The "Draft" Aggregate.** Identifying the stream boundaries. Is a "Draft" a single session of drawing one SVG?
- **Q5: Persistence of Progress.** Should progress be saved across app restarts for a single drawing?
- **Q6: Events.** Defining the core event set (e.g., `DraftStarted`, `GuidanceStepRevealed`, `DraftCompleted`).

### Section 3: Content & Assets
- **Q7: SVG Asset Preparation.** How are SVGs broken down into steps? Are they separate files, or one file with labeled groups/layers?
- **Q8: Asset Metadata.** Do we need a catalog of drawings (e.g., "Draw a Cat", "Draw a House")?

### Section 4: UX & Accessibility
- **Q9: Interaction for 4-year-olds.** Minimizing UI clutter. Should the "Next" button be the whole screen or a large footer?
- **Q10: Encouragement Loop.** How to handle the "I'm done" state? (Confetti, sound, etc.)
