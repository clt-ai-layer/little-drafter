## 2. Document Content Strategy

### Narrative End-State
I will describe a seamless interaction where a child picks a template (e.g., "Lion") and is guided through the drawing process. The narrative will emphasize the "Paper Support" aspect—the app isn't a drawing tool, but a guide. I'll describe the visual feedback (blue pulse) and the friendly TTS instructions that keep the child engaged and confident.

### Happy-Path Walkthrough
1. **Selection**: User picks a drawing from the catalog.
2. **Initialization**: `StartDraft` command is issued; state is initialized; first step reveals.
3. **Progression**: User draws on paper, then clicks the large green "Next" arrow. `ProgressStep` command; `DraftStepProgressed` event; next group pulses blue; TTS speaks.
4. **Completion**: User clicks "Next" on the last step. `DraftCompleted` event; celebration UI; Achievement projection updates badge count.

### Acceptance Checklist (Draft)
1. **[AC-01]** `StartDraft` creates a new Emmett session in the `in-progress` state.
2. **[AC-02]** Clicking the large green "Next" arrow increments `CurrentStepIndex` and reveals the next path group in the SVG.
3. **[AC-03]** The newly revealed SVG path group pulses in blue once.
4. **[AC-04]** The AudioService triggers TTS reading the `instruction` metadata for the current step index.
5. **[AC-05]** Clicking the large orange "Back" arrow decrements `CurrentStepIndex` (unless at 0).
6. **[AC-06]** Reaching the final step and clicking "Next" transitions the status to `completed` and emits `DraftCompleted`.
7. **[AC-07]** The `AchievementProjection` increments the user's star/badge count in MongoDB upon receiving `DraftCompleted`.

### Key Secondary Scenarios
- **Boundary Handling**: "Back" click at Step 0.
- **Robustness**: TTS fallback (visual guidance only).
- **Edge Case**: Rapid navigation clicks (ES consistency).

### Out-of-Scope
- On-screen drawing/tracing (the app is for paper support).
- Custom SVG uploads (catalog only for now).
- Multi-user drawing sessions.
