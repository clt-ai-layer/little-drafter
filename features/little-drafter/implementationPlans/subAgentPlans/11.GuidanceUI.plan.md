# Implementation Plan: Guidance UI

## Task 1: Plan (Phase 7 - Task 3)

This plan covers the implementation of the toddler-friendly drawing interface with progressive reveal and pulse cues.

### 1. Plan Details
- **Target Components**: `DrawingSession`, `NavigationBar`, `DrawingCanvas`.
- **Target Files**:
    - `packages/frontend/src/features/drawing/components/DrawingSession.tsx`
    - `packages/frontend/src/features/drawing/components/NavigationBar.tsx`
    - `packages/frontend/src/features/drawing/components/DrawingCanvas.tsx`
    - `packages/frontend/test/features/drawing/components/DrawingSession.test.tsx`
- **Patterns**: Next.js 15 App Router, Tailwind CSS, `framer-motion` for animations.

### 2. Scope Analysis
- **NavigationBar**: 
    - Bottom bar (`h-24`) with high-contrast buttons.
    - "Next" arrow: Green (`bg-green-500`), large icon.
    - "Back" arrow: Orange (`bg-orange-500`), large icon.
    - Dispatch commands: `ProgressStep` and `UndoStep`.
- **DrawingCanvas**:
    - Renders SVG paths from `CatalogService`.
    - Visibility logic: `opacity: 1` if `step.index <= currentStepIndex`, else `opacity: 0`.
    - Pulse animation: Use `framer-motion` to pulse the group where `step.index === currentStepIndex` in blue (`#0000FF`).
- **DrawingSession**:
    - Main container coordinating state and components.
    - Uses `useDraftSession` hook (mocked or from domain if ready).

### 3. Key Considerations
| Area | Consideration |
| --- | --- |
| **UX** | Kid-friendly: large targets, vibrant colors, clear visual feedback. |
| **Animations** | `framer-motion` pulse should be subtle but noticeable (e.g., `scale: [1, 1.05, 1]`). |
| **Testing** | Ensure `DrawingSession.test.tsx` covers navigation clicks and visibility logic. |
| **Next.js 15** | Ensure 'use client' directives are used for interactive components. |

### 4. Per-Output Content Plan
- **NavigationBar.tsx**: Tailwind-styled bar with Lucide icons (Chevrons).
- **DrawingCanvas.tsx**: Functional component mapping over SVG groups.
- **DrawingSession.tsx**: State management and layout assembly.
- **DrawingSession.test.tsx**: RTL tests for buttons and SVG visibility.

### 5. Implementation Steps
1. Create directory structure: `packages/frontend/src/features/drawing/components`.
2. Implement `NavigationBar.tsx`.
3. Implement `DrawingCanvas.tsx` with `framer-motion`.
4. Implement `DrawingSession.tsx`.
5. Implement `DrawingSession.test.tsx`.
6. Update `master.checkmarks.md`.
