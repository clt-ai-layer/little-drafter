# 12. Audio Implementation Plan

## DocRouter Findings / Scope Analysis
- **Bounded Context**: Frontend (`packages/frontend`).
- **Integration Surface**: `DrawingSession.tsx` component.
- **Patterns**: React hooks for side effects, singleton service for global browser APIs.
- **Constraints**: Web Speech API (`window.speechSynthesis`) is browser-dependent. Needs graceful degradation.

## Key Considerations Table
| Consideration | Strategy |
| ------------- | -------- |
| API Support | Check `window.speechSynthesis` before use. Log warning if missing. |
| Overlapping Speech | Call `cancel()` before starting new speech to prevent queuing build-up. |
| Kid-Friendly Tone | Set `rate` to ~0.9 or 1.0. Try to find a clear, friendly voice. |
| Cleanup | Ensure `cancel()` is called on component unmount. |
| Testability | Mock `speechSynthesis` and `SpeechSynthesisUtterance` in Jest. |
