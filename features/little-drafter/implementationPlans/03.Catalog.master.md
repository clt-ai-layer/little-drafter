## 1. Feature Overview
- **Name**: Catalog Service
- **Goal**: Serve SVG templates and extract step-by-step metadata.
- **Complexity**: Simple
- **Dependencies**: None

---

## 2. Implementation Phases

### Phase 1: Template Metadata Parsing
- Implement logic to load SVG files from a designated directory/storage.
- Use a library (e.g., `jsdom` or regex if simple) to find `<g>` tags with `data-step`.
- Extract `data-instruction` and `data-step` into a structured `DrawingTemplate` object.

### Phase 2: Catalog API
- Implement `getTemplate(id)` method.
- Return path data, instructions, and total step count.

## 4. Unit Tests

### Test File: `packages/backend/test/Catalog/Application/CatalogService.test.ts`

**Component Under Test**: `CatalogService`

#### Test Groups:

##### Template Loading
- **Given**: A valid SVG file with `data-step` attributes. **When**: `getTemplate`. **Then**: Returns correct number of steps and instructions.
- **Given**: An invalid template ID. **When**: `getTemplate`. **Then**: Returns `Result.fail` (NotFound).

##### Metadata Extraction
- **Given**: SVG with mixed groups. **When**: Parsing. **Then**: Only extracts groups tagged with `data-step`.
- **Given**: Missing `data-instruction` on a step. **When**: Parsing. **Then**: Provides a default fallback string.

**Mocks Required**: File system mock or temporary test SVG assets.
