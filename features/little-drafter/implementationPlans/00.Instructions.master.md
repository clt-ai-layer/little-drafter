### Key Patterns Discovered

1. **Vertical Slice Pattern**: Feature is organized as a complete slice from Domain (Decider) to Application (Command Bus) to UI (Next.js).
2. **"Paper Support" Guidance**: Pivot from on-screen tracing to acting as a visual/audio guide for drawing on real paper.
3. **Emmett Decider**: Pure functional state management using `decide` and `evolve` functions.

## Part 2: Requirement Understanding

**Name**: little-drafter
**Type**: Feature Implementation (Full Stack)
**What We're Building**: 
A craft application that assists 4-year-olds in drawing. The app reveals SVG parts progressively on a screen while the child replicates the lines on paper. 
- **Guidance**: Blue pulsing line on the latest step + TTS verbal instructions.
- **Navigation**: Large, colorful "Next" and "Back" arrows in a dedicated bottom bar.
- **Reward**: Success event on completion triggering an achievement (Stars/Badges).

## Part 3: Detailed Feature Breakdown

### Feature 1: Draft Domain (Emmett Decider)
**Purpose**: Manage the state and transitions of a drawing session.
**File**: `packages/backend/src/Draft/Domain/DraftDecider.ts`
**Changes**: Implement `initialState`, `decide()`, and `evolve()`.
- **States**: `TemplateId`, `CurrentStepIndex`, `Status` (`in-progress`, `completed`).
- **Events**: `DraftStarted`, `DraftStepProgressed`, `DraftStepUndone`, `DraftCompleted`.
**Complexity**: Moderate (ES logic)

### Feature 2: Drawing Catalog Service
**Purpose**: Serve SVG templates with metadata-tagged path groups.
**File**: `packages/backend/src/Catalog/Application/CatalogService.ts`
**Changes**: Load SVGs from disk/DB; extract group metadata (TTS text, sequence index).
**Complexity**: Simple

### Feature 3: Navigation UI (Frontend)
**Purpose**: Toddler-friendly controls and drawing canvas.
**File**: `packages/frontend/src/features/drawing/components/DrawingSession.tsx`
**Changes**: 
- Implement `NavigationBar` with large Pulsing Next Arrow (Green) and Back Arrow (Orange).
- SVG rendering logic with Blue Pulse animation on the latest path group.
**Complexity**: Moderate (Animation & UX)

### Feature 4: Audio Guidance (TTS)
**Purpose**: Read step-by-step instructions.
**File**: `packages/frontend/src/features/drawing/services/AudioService.ts`
**Changes**: Integration with Web Speech API (TTS).
**Complexity**: Simple

### Feature 5: Achievement Projection
**Purpose**: Track completed drawings.
**File**: `packages/backend/src/Achievement/Projections/AchievementProjection.ts`
**Changes**: Listen for `DraftCompleted` and increment user badge counts in MongoDB.
**Complexity**: Simple (Standard Projection)
